# Security Policy

## Supported Versions

This project is pre-1.0. Until a stable release exists, security support tracks
the current `main` branch only.

## Reporting a Vulnerability

Please do not open a public issue with exploit details. Use GitHub private
vulnerability reporting or a GitHub Security Advisory when available. If neither
is available, open a minimal public issue asking the maintainer for a private
reporting channel, without including exploit instructions or sensitive details.

Reports should include:

- Affected version or commit
- Reproduction steps
- Impact and likely attacker capability
- Any suggested fix or mitigation

The maintainer should acknowledge reports within 7 days and share a triage
decision or remediation plan within 30 days when practical.

## Repository Protection

Changes to the default branch should flow through pull requests unless the
repository owner intentionally uses an admin bypass during early development.
Pull requests are expected to pass CI and receive Code Owner review before
merge.
